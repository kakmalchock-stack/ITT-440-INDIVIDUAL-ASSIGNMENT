2. Background & Importance of Performance Engineering

Performance engineering ensures that applications remain stable, responsive, and reliable under various load conditions. Modern systems must handle global scale, unpredictable surges, and real-time processing—making performance testing essential.

3. Concepts of Modern Performance Testing

This section covers key terminology such as response time, throughput, concurrency, scalability, saturation points, tail latency, and bottlenecks, all of which help shape the interpretation of the system’s behavior under load.

4. Tool Selection Justification

K6 offers portability, JSON output, script-based flexibility, and excellent CI/CD compatibility, making it ideal for structured performance testing and analysis.

5. Test Environment Setup

The testing was conducted on Windows 10 using the portable version of K6. The execution command used was:

"C:\Program Files\k6\k6.exe" run testfile.js --out json=output.json

6. Detailed Test Methodology

6.1 Breakpoint Test

A gradual ramp-up of virtual users to detect the point where the system begins to degrade.

6.2 Capacity Test

A long-duration test performed to find the maximum stable throughput the system can sustain.

6.3 Ramp-Up Test

A progressive increase in traffic to evaluate scalability under growing demand.

8. Deep Performance Interpretation

The system performs well under small loads but collapses at higher concurrency levels. Latency spikes, instability, and failures indicate architecture bottlenecks.

9. Root-Cause Bottleneck Analysis

Likely causes include CPU saturation, database query latency, missing caching mechanisms, inadequate thread pools, and lack of load balancing.

10. Engineering Recommendations

  • Introduce caching (Redis, CDN).
  • Horizontal scaling and load balancing.
  • Optimize backend queries and increase connection pools.
  • Implement rate limiting and autoscaling.

11. Risk Analysis & Real-World Impact

Performance failures may lead to outages, revenue loss, degraded user experience, and operational instability.

12. Scalability Discussion

The system currently appears limited by vertical scaling constraints. Horizontal scaling is recommended for production workloads.

13. SLA/SLO/SLI Implications

Based on observed latencies and instability, the system is not currently able to meet standard performance SLAs under moderate load.

14. Limitations

The test target is a demo website and not production-grade infrastructure. Additional backend metrics would enhance analysis.

15. Future Work

Potential future enhancements include distributed load testing, spike analysis, chaos engineering, and real-user flow simulations.

16. Final Conclusion

The system demonstrates acceptable performance at low concurrency but fails under heavy load. K6 proved to be an effective tool for identifying performance thresholds, bottlenecks, and scalability issues.
